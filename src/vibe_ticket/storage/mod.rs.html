1//! Storage layer for vibe-ticket
2//!
3//! This module provides implementations for persisting and retrieving ticket data.
4//! It supports multiple storage backends through a common trait interface.
5//!
6//! # Storage Backends
7//!
8//! The storage module supports various backends:
9//! - File-based (YAML files for simple use cases)
10//! - `SQLite` (for local storage) - feature gated
11//! - `PostgreSQL` (for production deployments) - feature gated
12//! - In-memory (for testing)
13//!
14//! # Architecture
15//!
16//! The storage layer implements the repository pattern:
17//! - Repository traits define the interface
18//! - Concrete implementations for each storage backend
19//! - Migration support for schema management
20//! - Connection pooling and transaction management
21//!
22//! # Concurrent Access Protection
23//!
24//! The file storage implementation includes built-in protection against concurrent
25//! modifications through a file-based locking mechanism:
26//!
27//! - **Automatic Locking**: All write operations acquire exclusive locks
28//! - **Lock Files**: Created as `<filename>.lock` with metadata
29//! - **Retry Logic**: Operations retry up to 10 times with 100ms delays
30//! - **Stale Lock Cleanup**: Locks older than 30 seconds are removed automatically
31//! - **RAII Pattern**: Locks are released automatically using Rust's Drop trait
32//!
33//! This ensures data integrity even when multiple users or processes access
34//! tickets simultaneously.
35//!
36//! # Example
37//!
38//! ```ignore
39//! use vibe_ticket::storage::{FileStorage, Repository};
40//!
41//! // Initialize storage backend
42//! let storage = FileStorage::new(".vibe-ticket");
43//!
44//! // Use storage through repository traits (locking is automatic)
45//! let tickets = storage.get_all()?;
46//! ```
47//!
48//! # Error Handling
49//!
50//! All storage operations return `Result<T, VibeTicketError>` to handle:
51//! - I/O errors
52//! - Serialization/deserialization errors
53//! - Not found errors
54//! - Permission errors
55//! - Lock acquisition failures
