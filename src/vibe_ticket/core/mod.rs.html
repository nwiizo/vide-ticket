1//! Core business logic for vibe-ticket
2//!
3//! This module contains the fundamental business logic and domain models for the ticket
4//! management system. It is independent of any specific storage or presentation layer.
5//!
6//! # Architecture
7//!
8//! The core module follows Domain-Driven Design principles:
9//! - Domain entities (Ticket, Project, User, etc.)
10//! - Value objects (`TicketId`, Status, Priority, etc.)
11//! - Domain services and business rules
12//! - Repository traits (interfaces for storage)
13//!
14//! # Example
15//!
16//! ```no_run
17//! use vibe_ticket::core::{Ticket, Status, Priority};
18//!
19//! let mut ticket = Ticket::new("fix-login-bug", "Fix login bug");
20//! ticket.description = "Users cannot login with special characters".to_string();
21//! ticket.priority = Priority::High;
22//! ticket.start();
23//! ```
24//!
25//! # Design Principles
26//!
27//! - The core module should have no dependencies on external crates for business logic
28//! - All I/O operations should be abstracted through traits
29//! - Business rules should be enforced at this layer
30
